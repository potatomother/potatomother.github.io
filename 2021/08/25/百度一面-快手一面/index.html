6.this指向？如何改变this指向

this指向分成两类，一个是箭头函数的this，另一个是普通函数的this。

1.箭头函数本身没有this，使用外层函数的this

2.普通函数的this，如果有new 对象，this指向new的新对象，

​ 谁调用函数，函数中的this就只想这个对象

​ 直接调用函数，this指向window

改变this指向，call apply bind

call 确定参数个数 fun.call（argthis，arg1,arg2…）

apply不确定参数个数 fun.apply（argthis，[arg1,arg2..]）

bind 参数和call一样，返回值是函数的拷贝

1.跨域方案

什么是跨域

域名地址的协议 域名 端口都需要相同，如果存在不同则是跨域。即使不同的域名指向的ip地址是相同的也是属于跨域。

同源策略限制的内容是什么呢？

  • Cookie、LocalStorage、IndexedBD等存储性内容
  • DOM节点
  • AJAX请求发送之后，浏览器会拦截返回结果

那有什么是不限制的呢？

  • <img src=...>
  • <link herf=...>
  • <script src=..>

有什么跨域方案？

1.JSONP，原理？

利用了script不受同源策略的限制，

缺点：只能使用get，容易受到XSS

2.CROS，跨域资源共享

当时使用XMLHttpRequest发送请求的时候如果浏览器发现违反了同源策略，会自动加上一个请求头。

后端收到请求之后会在Respose Headers 中加入一个属性Access-Control-Allow-Origin；

浏览器判断响应中的Access-Control-Allow-Origin值是否和当前地址相同，匹配成功继续相应处理。

缺点：忽略cookie，对浏览器版本有一定要求

3.代理方案跨域：

正向代理 反向代理（nginx）

4.websocket 协议

是html5的新协议，可以做到浏览器于服务器的全双工通信，允许跨域请求

缺点：对浏览器版本有一定要求，需要服务器指出websocket协议。

前端路由都有什么模式？分别说说

前端路由是一个前端不同页面的状态管理器，他可以不向后台发送请求而直接通过前端技术实现多个页面的效果

哈希模式

HTML页面中通过锚点定位原理可以进行无刷新跳转，触发后url地址中会多出#+ ‘XXX’ 部分,在全局的window对象上触发hashChange事件，这样在页面锚点哈希改变为某个预设值的时候，通过代码触发对应的页面DOM改变，就可以实现基本的路由。同时，hash值的改变都会在浏览器的访问历史中增加一个记录。因此可以通过浏览器的回退、前进按钮控制hash的切换

  • hash 是 URL 中 hash (#) 及后面的那部分，常用作锚点在页面内进行导航，改变 URL 中的 hash 部分不会引起页面刷新
  • 通过 hashchange 事件监听 URL 的变化，改变 URL 的方式只有这几种：通过浏览器前进后退改变 URL、通过<a>标签改变 URL、通过window.location改变URL，这几种情况改变 URL 都会触发 hashchange 事件

历史模式

HTML5的History API 为浏览器全局history对象增加的扩展方法。一般用来解决ajax请求无法通过回退按钮回到请求前状态的问题。

当浏览器访问一个页面时，当前地址的状态信息会被压入历史栈，当调用history.pushState()方法向历史栈中压入一个新的state后，历史栈顶部的指针指向新的state的。

History.replaceState()更新历史栈上最新的入口。

  • history 提供了 pushState 和 replaceState 两个方法，这两个方法改变 URL 的 path 部分不会引起页面刷新
  • history 提供类似 hashchange 事件的 popstate 事件，但 popstate 事件有些不同：通过浏览器前进后退改变 URL 时会触发 popstate 事件，通过pushState/replaceState或<a>标签改变 URL 不会触发 popstate 事件。好在我们可以拦截 pushState/replaceState的调用和<a>标签的点击事件来检测 URL 变化，所以监听 URL 变化可以实现，只是没有 hashchange 那么方便。
